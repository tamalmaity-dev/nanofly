# VERSION: v0.3.16.4-beta

# Walkthrough - Alphanumeric Passwords & Robust Connection String Regex Parsing

This version addresses critical connection string and URL parsing issues caused by special characters (specifically `#`) in auto-generated database passwords.

## Changes Made

### 1. Safe Alphanumeric Database Passwords
- **Alphanumeric Character Set**: Updated the password generation utility (`generateSecurePassword` in [password.js](file:///e:/All%20Coding%20Projects/Production%20Projects/NanoFly/web/src/utils/password.js)) and the React frontend password generator (`generatePassword` in [ProjectDetail.jsx](file:///e:/All%20Coding%20Projects/Production%20Projects/NanoFly/web/src/pages/ProjectDetail.jsx)) to generate passwords containing only alphanumeric characters (`a-z`, `A-Z`, `0-9`). 
- **Security Check**: Maintained strong password security by standardizing the generated password length to 24 characters.
- **Escape-Free Connections**: Alphanumeric passwords prevent breaking URL query/fragment parsers, terminal commands, or application environment configurations (such as WordPress database configuration files).

### 2. Robust Connection String Regex Parser
- **URL Parser Fallback**: Replaced the native `new URL()` parser in the frontend **ConnectionDetailsPanel** (`ProjectDetail.jsx`) with a robust regex pattern matcher (`^([a-z0-9]+):\/\/([^:]+):(.*)@([^:]+):([0-9]+)\/(.*)$`).
- **Special Characters Support**: This ensures that even if users manually add special characters (such as `#`, `@`, `$`, or `/`) to their database passwords, the frontend can correctly parse and display the Access Credentials (host, port, database name, username) without any fallback mismatch.
