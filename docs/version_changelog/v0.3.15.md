## 🚀 Key Features & Enhancements

### 1. Host VPS/Server Reboot API & Controls
- **VPS/Server Reboot Endpoint**: Implemented `POST /api/v1/settings/reboot` inside [server.go](file:///e:/All%20Coding%20Projects/Production%20Projects/NanoFly/internal/server/server.go) which triggers a host-level system reboot (`sudo reboot` / `reboot` on Unix/Linux, `shutdown /r /t 0` on Windows) asynchronously.
- **Danger Zone UI**: Added a dedicated "Danger Zone" segment in the Settings page's System Tab in [Settings.jsx](file:///e:/All%20Coding%20Projects/Production%20Projects/NanoFly/web/src/pages/Settings.jsx) with a red **Reboot Server** action button, complete with confirmation flows, loading states, and health polling to auto-reload the page when back online.

### 2. Premium Service Monitoring UI
- **Glassmorphism Redesign**: Upgraded stat cards with rich glassmorphism backdrops, subtle glowing borders, and smooth transition animations in [MonitoringPanel.jsx](file:///e:/All%20Coding%20Projects/Production%20Projects/NanoFly/web/src/components/panels/MonitoringPanel.jsx).
- **Responsive Resource Charts**: Restyled CPU, Memory, Disk, and Network charts with glowing tooltips, smooth SVG path gradients, and clean layout geometry.
- **Micro-Animations**: Added active pulsing indicators for live monitoring status badges.

### 3. File Manager Editor Upgrades
- **VS Code Theme Styling**: Overrode syntax tokens to render classic VS Code Dark+ theme syntax highlighting.
- **Line Numbers Gutter**: Implemented a responsive line numbers gutter with exact matching line-height alignment to prevent offset issues on all devices in [CodeEditor.jsx](file:///e:/All%20Coding%20Projects/Production%20Projects/NanoFly/web/src/components/CodeEditor.jsx).
- **Graceful Rendering**: Added try-catch syntax parsing safety blocks to prevent React Error 130 and other crashes from unescaped text inputs.

## 🛠️ Critical Bug Fixes

### 1. Update Downgrade Infinite Reload Fix
- **Loop Prevention**: Modified `handleUpdateLog` to automatically clear the `"done"` status back to `"idle"` upon first read. This successfully prevents infinite reloads when checking for updates/downgrades on restart.
- **Channel Sync**: Configured the Release Channel settings to persist immediately on change to prevent database mismatches upon system reboot.

### 2. Database Connection Port Binding & Auto-Detection Fix
- **Host Binding**: Changed database container host port binding `HostIP` from `127.0.0.1` to `0.0.0.0` in [manager.go](file:///e:/All%20Coding%20Projects/Production%20Projects/NanoFly/internal/api/docker/manager.go) to allow container-to-container connection (such as WordPress setup) via the host gateway loopback address (`host.docker.internal`).
- **Auto-Credentials/Port Detection**: Enhanced the WordPress env enrichment logic to automatically look up database container ports and credentials within the same project, auto-mapping `WORDPRESS_DB_HOST`, `WORDPRESS_DB_USER`, `WORDPRESS_DB_PASSWORD`, and `WORDPRESS_DB_NAME` dynamically.

### 3. Security Vulnerability Remediations
- **Secure Script Execution**: Fixed the suspicious `curl | sh` execution pattern in the installation script [install.sh](file:///e:/All%20Coding%20Projects/Production%20Projects/NanoFly/install.sh), utilizing secure file-downloading methods.
