# NanoFly v0.3.20.11-beta — Custom Domain SSL, Unmounted Drive Auto-Mount & Dashboard Storage Cards

**Release Date:** 2026-05-30  
**Type:** Feature & Enhancement Update  
**Urgency:** High  

---

## Summary

This release introduces native custom domain activation for the NanoFly panel with automatic Traefik reverse proxy routing and Let's Encrypt SSL. It also adds intelligent unmounted external drive auto-mounting on Linux alongside a new Dashboard visualization section for all server storage drives.

## Key Changes

### 1. Panel Custom Domain & Automatic SSL Routing
- **Traefik Dynamic Configuration**: Added a backend endpoint `POST /api/v1/settings/panel-domain` that automatically writes dynamic Traefik yaml configurations (`panel.yml`) under the monitored `/etc/traefik/dynamic` directory on host.
- **Reverse Proxy Routing**: Configures the panel to be accessible via any user custom domain over port 443 (`websecure` entrypoint) mapping to `host.docker.internal:<panel_port>` on the host.
- **Auto Let's Encrypt SSL**: Provisions free SSL certificates using Traefik's `letsencrypt` ACME TLS challenge resolver.
- **Sanitized Domain Inputs**: Automatically cleans user URL submissions to strip protocols (`http://`, `https://`), ports, and trailing directories before validation.
- **DNS Record Lookup**: Runs validation checks on the target custom domain A-record against server interface IPs, reporting friendly diagnostic outputs on DNS mismatch while remaining flexible.

### 2. Premium Settings Page UI
- **Activation controls**: Replaced the plain "Panel URL" input field with a professional **Panel Custom Domain & SSL** section under the General tab.
- **Interactive States**: Added visual loaders, success confirmation cards for active SSL routing, DNS prerequisite warnings, and an instant **Deactivate** button to revert reverse proxy routing.

### 3. Unmounted External Drive Auto-Mounting (Linux)
- **Partition Auto-Mounting**: Upgraded physical drive discovery to execute `lsblk` with JSON formatting to locate physical partitions (like `sda4`, `sda7`) that have valid filesystem types but are currently unmounted.
- **Zero-Config Mounts**: Automatically creates target directories under `/mnt/nanofly-<partition_name>` and mounts the partition, making plugged-in external drives instantly accessible in the File Manager sidebar for all file operations (listing, creating, editing, uploading, deleting) without manual terminal commands.

### 4. Overview Dashboard Drives Section
- **Live Disk Metrics**: Added the full list of detected system and external drives (populated dynamically) to the Metrics WebSocket snapshot.
- **Drives & Storage Widget**: Added a dedicated dashboard section visualizing all drives with usage details, mounted directories, and progress bars color-coded by capacity safety levels (green, yellow warning at 75%, and red danger at 90%).
