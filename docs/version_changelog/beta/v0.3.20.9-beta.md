# NanoFly v0.3.20.9-beta — Selective Docker Resource Pruning

**Release Date:** 2026-05-30  
**Type:** Feature Update  
**Urgency:** Medium  

---

## Summary

This release introduces selective Docker resource pruning in the Settings page. Users can now choose which Docker resource categories (Stopped Containers, Unused Images, Unused Volumes, Unused Networks) they want to clean up using checkboxes in a card-based UI, rather than running a complete system prune unconditionally.

## Key Changes

### 1. Selective Pruning UI (Frontend)
- Added checkboxes for **Stopped Containers**, **Unused Images**, **Unused Volumes**, and **Unused Networks** in the Docker Cleanup section of the System Tab.
- Designed a card-based layout featuring hover transitions, modern rounded card items, and accent indicator borders for selected categories.
- Toggling the checkboxes allows users to specify which items are targeted by the prune action.
- Updated the prune confirmation dialog to dynamically show the list of selected resource categories (e.g. `containers, images`).
- Checks that at least one resource category is selected before executing the operation.

### 2. Selective Pruning Backend Integration
- Updated API client call and backend handler to transmit and process prune selection parameters (`containers`, `images`, `volumes`, `networks`).
- The backend selectively calls prune functions for each category depending on the request payload.
