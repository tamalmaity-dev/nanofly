# NanoFly v0.3.20.15-beta — Usability Fixes, Theme Resiliency, Form Input Protection, Configuration Renaming & Deployment Domains

**Release Date:** 2026-05-31  
**Type:** UI Enhancements, Usability Fixes & Resiliency Update  
**Urgency:** High  

---

## Summary

This release resolves several visual clutter points, button misalignments, editor focus ring issues, form input reset bugs, and Node.js builder template crashes. The File Manager grid cards are now cleaner with hover-only action controls, while editing forms are protected from background refresh overrides. The Node.js build pipeline is now fully resilient, fallback-capable, and lockfile-independent. Additionally, the "Settings" tab has been renamed to "Configuration" throughout the interface, and custom domain configuration support has been added directly to the first-time app deployment creation page.

## Key Changes

### 1. Hover-Only Overlay Actions in Grid View
- **Clutter Elimination**: Action icons (ZIP/unzip, rename, copy path, delete) on folder/file cards in Grid View are hidden by default and fade in smoothly on card hover via absolute-positioned overlays.
- **Persistent Metadata**: Size information is placed directly below the folder/file names so it remains visible at all times.

### 2. Editor Focus Border Elimination
- **Focus Ring Reset**: Stripped browser-default focus outlines or white/grey focus borders on the `CodeEditor` textarea and pre element tags, keeping the developer environment visually clean when typing.

### 3. Header Action Alignments
- **Vertical Alignment**: Added `border: 1px solid transparent` on the Save button inside the editor header to match the height and vertical padding of the adjacent Close button, aligning them perfectly.

### 4. Vibrant Warning Themes
- **Vibrant Status Colors**: Restyled warning panels to render high-contrast yellow (`var(--yellow)` / `var(--yellow-dim)`) for the Safe Workspace status, and bright red for System File alerts.

### 5. Persistent & Independent Editor Themes
- **Theme Isolation**: Swapped background/text variables to use static mappings for Tomorrow, Dracula, Monokai, and GitHub Light. The code editor themes and light/dark modes remain fully independent of the global web app theme state.
- **Theme Persistence**: Choices are saved directly to local storage (`nanofly_editor_theme`, `nanofly_editor_dark`) so preferences persist across file switching and browser refreshes.

### 6. Fullscreen Workbench Preview Dialog
- **Expanded Dialogue View**: Added a "Maximize" option next to Close/Save in the editor preview. Clicking it renders the entire media/code/editor workspace in a massive Radix UI modal dialog (`82vh` height / `95vw` width).

### 7. Form Inputs Refresh Protection (Settings Polling Pause)
- **Polling Paused on Edit**: Modified the parent service polling hook to pause 5-second background interval requests while the user is actively on any editing tabs: Settings (`settings`), Resource Limits (`resources`), and Environment Variables (`envvars`). This prevents the state from resetting and wiping user inputs while typing.

### 8. Node.js Deployment Fallback Resiliency
- **Lockfile-Independent Install**: Replaced the default Node.js build command `npm ci --omit=dev` with a resilient shell fallback check:
  ```bash
  if [ -f package-lock.json ]; then npm ci --omit=dev; else npm install --omit=dev; fi
  ```
- This dynamically leverages `npm ci` for speed if `package-lock.json` is present, while safely falling back to standard `npm install` if the lockfile is missing, resolving deployment build crashes.

### 9. Settings renamed to Configuration
- **Tab Renaming**: Renamed the "Settings" tab in the service view layout to "Configuration" throughout the UI.
- **Save Labeling**: Updated toast messages, success indicators, save actions, and headers to refer to "Configuration" instead of "Settings" (e.g. `Save Configuration`, `Configuration saved. Redeploy to apply.`, etc.).

### 10. Creation Domain field
- **First-Time Domain Config**: Added a custom "Domain" input field inside the "Basics" configuration section during first-time service deployment.
- **Auto-population**: Automatically populates the input field with a pre-configured random routing domain (e.g., `[rand].localhost.sslip.io`) for easy review or custom configuration.
- **Domain Mapping**: connected the custom domain configuration to the service creation pipeline, auto-registering the domain on successful deployment.
