# NanoFly v0.3.20.14-beta — Borderless File Manager Overhaul, Rename Actions & Interactive Image Editor

**Release Date:** 2026-05-30  
**Type:** Feature, Theme Support & Visual Redesign Update  
**Urgency:** High  

---

## Summary

This release introduces a major visual overhaul, usability optimization, and feature expansion for the NanoFly File Manager. The interface now features borderless panels separated by subtle divider rules in a clean developer IDE column layout. Hardcoded dark colors are fully replaced with standard CSS theme variables to fix light-mode compatibility. Additionally, we added Drag-and-Drop uploads, folder-first sorting, folder/file renaming, support for a wider range of media formats, native media asset loading indicators, and an interactive Image Editor that supports scroll wheel zooming, slider zooming, click-drag panning, and realtime style adjustments.

## Key Changes

### 1. Theme Compatibility (Light/Dark Mode Fix)
- **Responsive Colors**: Replaced all hardcoded background and text values in the file manager with theme CSS variables (`var(--bg-base)`, `var(--bg-surface)`, `var(--bg-elevated)`, `var(--border)`, `var(--text-primary)`, etc.).
- **Code Editor Syntax**: Integrated light-theme syntax overrides for Prism highlight tokens, enabling high-contrast code readability across theme modes.
- **Code Editor Background**: Overrode the hardcoded dark background of syntax highlighters with `var(--bg-surface)` (which is a premium dark blue-grey in dark mode, not pure black) to make it integrate beautifully with panels.

### 2. Seamless Borderless Columns Divider Layout
- **IDE Column Layout**: Stripped individual container borders, box shadows, and margins to build a unified column system divided only by vertical separator rules (`1px solid var(--border)`).
- **High-Density Sidebar**: Restyled drives and storage mounts to align with file explorer lists.

### 3. Horizontal Column Sliding & Text Size Improvements
- **Column Constraints**: Applied minimum width restrictions (Drives Sidebar: `180px`, Directory Explorer: `300px`, Workbench Editor/Media: `450px`) combined with `flex-shrink: 0` constraints.
- **Horizontal Scroll**: Set `overflow-x: auto` on the columns wrapper, allowing columns to slide smoothly on smaller windows without collapsing layout content.
- **Enlarged Text**: Increased font sizes of all key items (labels, file lists, details, warnings, headers, code text) to 14px/0.88rem/0.95rem for clear, comfortable reading.
- **Consistent Alignment**: Standardized action buttons inside the file manager header to align perfectly.

### 4. System Disk Partition Filtering
- **System Partitions Skipped**: Updated the Unix drives collector (`drives_unix.go`) to dynamically detect the root system disk partition (e.g. `sda1` or `mmcblk0p2`) and filter out all other partitions belonging to the same physical system drive (like boot, recovery, etc.) from being mounted or reported as "external" storage. Only distinct physical disks (like attached USB drives) are auto-mounted.

### 5. Drag-and-Drop File Uploads
- **Drop Zone Overlay**: Implemented HTML5 drag-and-drop support. Dragging files over the explorer list displays a drop overlay zone to upload multiple items in the background.

### 6. Folder-First Sorting Controls
- **Metrics Dropdown**: Sort folder lists dynamically by **Name**, **Size**, or **Last Modified Date** (Ascending/Descending).
- **Priority Directories**: Directory items are automatically prioritized and kept at the top of the sorted lists.

### 7. Files/Folders Renaming (API & UI Modal)
- **Rename Endpoint**: Registered `/api/v1/files/rename` on the Go backend using standard `os.Rename`.
- **Rename Dialog**: Added inline Rename row buttons that trigger a clean prompt input modal to submit rename requests.

### 8. Interactive Image Editor
- **Multi-modal Zoom**: Scroll over the canvas to zoom in/out smoothly, or use the integrated Zoom slider bar and +/- buttons with precise percentage indicators.
- **Drag Panning**: Click and drag zoomed images in any direction.
- **Grid Filter Adjustments**: Arranged adjustment sliders (Brightness, Contrast, Grayscale, Sepia), Invert checkbox, Rotate 90°, and Reset controls in a uniform, professional grid panel.

### 9. Expanded Media Formats & Native Loaders
- **Wide Extensions Support**: Enabled native playback recognition for a broader range of media:
  - **Images**: `.heic`, `.avif`, `.webp`, `.tiff`
  - **Audio**: `.m4a`, `.aac`, `.wma`
  - **Video**: `.mov`, `.mkv`, `.flv`, `.wmv`
- **Native Loading Indicators**: Hooked state-driven `mediaLoading` spinner overlays to native player events (`onLoad` for images, `onCanPlay` for audio/video) to display loading feedback during asset fetching.
