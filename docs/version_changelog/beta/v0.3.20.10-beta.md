# NanoFly v0.3.20.10-beta — WordPress Container Initialization & File Manager Drives Sidebar

**Release Date:** 2026-05-30  
**Type:** Feature & Bug Fix Update  
**Urgency:** High  

---

## Summary

This release resolves a deployment completion reporting issue for WordPress app services and adds physical mounts and Windows logical drives listing into the File Manager.

## Key Changes

### 1. WordPress Container Initialization Check
- **Readiness Wait Loop**: Added container log monitoring for WordPress services. The deployment status now stays in `deploying`/`building` while WordPress copies its core files and configures the database connection inside the container.
- **Web Server Ready Detection**: The deployment only transitions to `running` / `complete` once the logs confirm that Apache or PHP-FPM has fully started and is accepting connections, preventing false-positive "Running" status while the service is still initializing.

### 2. File Manager Drives & Mounts Sidebar
- **Mount Points Discovery**: Added a `/drives` API endpoint that discovers and queries physical drives (Unix device nodes like USB drives under `/media/` or `/mnt/`, and Windows logical drive letters like `C:\`, `D:\`).
- **Sidebar Selector**: The File Manager UI now displays a Left Sidebar listing all detected drives, along with real-time free space metrics and clean hard drive icons.
- **Easy Navigation**: Users can click any drive to jump directly to its mount path.
