# VERSION: v0.3.16.7-beta

# Walkthrough - Docker Container Links, Compatibility, & Update Panel Crash Fixes

This version resolves the persistent "Error establishing a database connection" in WordPress and git-deployed applications, and fixes the settings page crash on system updates.

## Changes Made

### 1. Robust Docker Container Links & Port Resolution
- **Link Mapping**: Enhanced the main image deployment (`DeployApp`) and git/local deployment flows to link database containers directly using container links (`nf-db-<name>:<name>`), allowing applications to communicate natively on port `3306` inside the Docker network.
- **Port Drift & Firewall Bypassing**: Replaced host gateway loopback fallback connections with native container links, bypassing host firewalls and ensuring stable database routing.

### 2. MySQL / MariaDB User Compatibility
- **Designated User Default**: Switched default WordPress environment user fallback from `"root"` to `"nanofly"` when `db_user` is empty, matching the default database user generated during database service creation and avoiding MySQL remote root login blocks.

### 3. DATABASE_URL Injection
- **Connection URL**: Injected `DATABASE_URL` environment variables in `enrichWordPressEnv` to support app components that utilize connection strings directly (`mysql://<user>:<password>@<host>/<dbname>`).

### 4. React Reference Error Fix
- **Fragment Imports**: Replaced `<React.Fragment>` with `<Fragment>` in the update progress steps component inside [Settings.jsx](file:///e:/All%20Coding%20Projects/Production%20Projects/NanoFly/web/src/pages/Settings.jsx) to prevent `Uncaught ReferenceError: React is not defined` in minified/production builds.
