# Enforce Unlimited Resource Tier for Database Containers

## Changes

### 1. Default Database Containers to Unlimited Resource Tier
- **Fix**: Updated `CreateDB` in [manager.go](file:///e:/All%20Coding%20Projects/Production%20Projects/NanoFly/internal/api/docker/manager.go) to always assign the `"unlimited"` resource tier to database containers, ignoring default service tiers (`micro`, `nano`, `standard`, `large`).
- **Overriding**: Custom limits defined explicitly by the user are still respected.
- **Benefit**: Fully prevents database containers (e.g. MySQL 8.0, PostgreSQL) from being terminated on startup or operation due to kernel Out-Of-Memory (OOM) killer constraints, ensuring high-availability database operation on low-resource environments (like Raspberry Pi 3).
