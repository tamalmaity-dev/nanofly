# VERSION: v0.3.16.3-beta

# Walkthrough - WordPress DB Setup Enhancements, Payload Parsing Fixes, and Database Deployment Logs

This version contains essential fixes and optimizations for database setups, managed database container logs, and the WordPress deployment workflow.

## Changes Made

### 1. Database Payload Parsing & Connection Mismatch Fix
- **Payload Decoding**: Fixed a bug in the database creation HTTP handler (`CreateDatabase` inside [handler.go](file:///e:/All%20Coding%20Projects/Production%20Projects/NanoFly/internal/api/services/handler.go)) where incoming `db_user`, `db_password`, and `db_name` variables were not being decoded from the JSON payload. This ensures correct parameters are persisted and used when launching the containers.
- **Port Mapping Reuse**: Ensured existing host database ports are preserved on re-deployments of database containers.

### 2. Managed Database Build & Setup Logs
- **Deployment Tracking**: Updated `CreateDatabase` in [service.go](file:///e:/All%20Coding%20Projects/Production%20Projects/NanoFly/internal/api/services/service.go) to register a database container setup process in the `deployments` history table and write operational step logs to it.
- **Live Logs Visibility**: Enabled the **Deployments** history and build logs panel inside [ProjectDetail.jsx](file:///e:/All%20Coding%20Projects/Production%20Projects/NanoFly/web/src/pages/ProjectDetail.jsx) for database-type services. Users can now view initialization logs (ports, username, engine) directly from the details panel.

### 3. Editable Auto-Generated Database Credentials & Real-time Env Sync
- **Customized Credentials**: Added UI input fields for **Database Name**, **Database Username**, and **Database Password** to the WordPress runtime configuration step in [AddServiceConfigFields.jsx](file:///e:/All%20Coding%20Projects/Production%20Projects/NanoFly/web/src/components/AddServiceConfigFields.jsx) (visible when selecting automatic database creation).
- **Auto-Fill Defaults**: Automatically populates clean defaults (`wordpress` username and database name, plus a randomly generated secure password).
- **Real-Time Environment variables Synchronization**: Added live `onChange` listeners so that any modifications made by the user in the editable DB credential inputs instantly update the corresponding values inside the **Environment variables** text box below. This prevents any confusion or mismatch during deployments.
- **Submit Mapping**: Updated the submission logic in [ProjectDetail.jsx](file:///e:/All%20Coding%20Projects/Production%20Projects/NanoFly/web/src/pages/ProjectDetail.jsx) to correctly apply the user-defined credentials to the new database service and pass them along to the WordPress environment variables.
