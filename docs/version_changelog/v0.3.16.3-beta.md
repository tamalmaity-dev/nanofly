Let's explain the synchronization and how the database connection is resolved:

### How the Database Connection & Env Sync works:

1. **Environment Variables Auto-Sync:**
   * When you edit the **Database Name**, **Database Username**, or **Database Password** input fields in the UI, they **instantly synchronize** in real-time with the **Environment variables** text area below.
   * As you type, the matching `WORDPRESS_DB_NAME`, `WORDPRESS_DB_USER`, and `WORDPRESS_DB_PASSWORD` lines are replaced automatically inside the text area.

2. **How the Database Connection is Resolved:**
   * **Host Connection:** The auto-created MySQL/MariaDB database is deployed in a Docker container. WordPress connects to it through `host.docker.internal:<host_port>` (which points to the docker host machine mapping the database container's port).
   * **Synchronous Port Resolution:** Once you click "Deploy Now", NanoFly synchronously requests a free host port from the Docker host, sets up the database container on that port, and injects that exact port into `WORDPRESS_DB_HOST` dynamically.
   * **Credentials Injection:** Even if there is any mismatch in the text area, the deployment handler uses the values from the **Database Credentials** input fields to create the database and injects those exact values as `WORDPRESS_DB_USER`, `WORDPRESS_DB_PASSWORD`, and `WORDPRESS_DB_NAME` directly into the WordPress container environment variables.
