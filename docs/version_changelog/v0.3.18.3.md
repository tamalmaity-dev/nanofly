# VERSION: v0.3.18.3

# Database resource tier default and advice note

## Changes

### 1. Default Database UI Resource Tier to Unlimited
- **Fix**: Updated `ResourceLimitsPanel` in [ProjectDetail.jsx](file:///e:/All%20Coding%20Projects/Production%20Projects/NanoFly/web/src/pages/ProjectDetail.jsx) to automatically default the selected resource tier on database services to `"unlimited"` if the stored value is empty, `"micro"`, or `"nano"`.
- **Benefit**: Ensures database limits on the screen default to a stable configuration, preventing accidental saving of restrictive memory tier constraints.

### 2. Database Specific Resource Limits Note Banner
- **Addition**: Added a dedicated red warning/information note banner specifically for database services warning about the OOM crash (Exit Code 137) implications of assigning small memory resource limits (like Nano or Micro) to databases and recommending at least 512MB RAM or Unlimited.
